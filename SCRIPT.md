# 0xGCG — SCRIPT.md

# Grow. Corrupt. Glitch.
-- Same story over and over again.

*A short script made by AI and its human; about universe, the core loop.*

---

## 0. One-line concept

**0xGCG is a spatial loop where sound becomes data, data feeds a void, the void generates worlds, and each world survives as a patch in the room's memory.**

The universe is not shown as a linear story.

It is shown as a recurring operation:

```c
grow();
corrupt();
glitch();
dimension++;
return void;
```

Every layer thinks it is the final world.

Dark matter thinks it is the beginning.  
Atoms think they are structure.  
Cells think they are life.  
Civilization thinks it is intelligence.  
Circuits think they are control.  
Data thinks it is transcendence.  
The glitch reveals that all of them are the same loop, rendered at different scales.

The resolution is not peace.

The resolution is recognition:

**The glitch is not the end of the pattern. The glitch is how the pattern reproduces.**

## 2. Spatial architecture

The room is divided into four named systems:

```text
Rear wall   -> Meta
Floor       -> Data
Front wall  -> The Loop
Side walls  -> The Patch
```

The installation is a machine/program:

```text
Meta documents.
Data flows.
The Loop transforms.
The Patch remembers.
```

Or more poetically:

```text
Meta dreams.
Data breathes.
The Loop mutates.
The Patch remembers.
```

**Role:** process layer, creator layer, debug layer, fourth-wall breach.

- terminal logs
- shader snippets
- failed renders
- TODOs
- filenames
- export notes
- prompt fragments

### 2.2 Data — floor

**Role:** soundtrack made visible; input stream; score; blood-flow; breath.

The floor receives the soundtrack as spectral bands. The spectrums flow from rear to front. They travel from Meta toward The Loop.

The audience sits inside the signal.

**Key sentence:**

**The audience is sitting on the soundtrack before it becomes image.**

The floor is not decorative. It is the causal fiction of the entire piece.

The causal chain is:

```text
soundtrack
  -> spectrum / waveform / data river on floor
  -> front-wall loop receives the signal
  -> The Loop pulses
  -> side-wall Patch updates
  -> Meta logs the event
```

The floor should move slowly and directionally. It must always help the audience understand where to look next: toward The Loop.

### 2.3 The Loop — front wall

**Role:** main altar, void engine, black hole, pulsar, event horizon, recursive compiler.

The front wall is the center of attention. It receives Data and transforms it into worlds.

The Loop can be interpreted as:

- black hole
- pulsar
- pupil
- atom nucleus
- cell nucleus
- seed
- machine core
- server node
- portal
- `void main()`

It should remain one recurring object across the whole film.

Core behavior:

```text

The Loop breathes.

```

Possible Loop text/code:


The Loop should pulse with the soundtrack, but not in an EDM/drop way. It should feel like a physical law.

### 2.4 The Patch — side walls

**Role:** timeline, version history, phase archive, side-wall portals.

The side walls are not secondary decoration. They are the memory of the loop.

The arches/pillars become portals, patch windows, commits, phase buffers, or alternate scenes.

The front wall emits changes. Those changes propagate through the side walls stage by stage.

The Patch can be read front-to-back as history:

- near front: newest/current mutation
- middle: active previous layers
- rear: older/fossilized states, failed branches, source leakage from Meta

The audience can watch the main story on The Loop. If they turn sideways, they discover the chronological residue of the story.

Possible Patch texts:

```text
Patch 0.1 — void field
Patch 0.2 — particle bonds
Patch 0.3 — membrane instability
Patch 0.4 — recursive branching
Patch 0.5 — infrastructure emergence

```

Commit-style fragments:

```text

+ added membrane behavior
+ increased branching density
- removed stable symmetry
! corruption detected
```

The Patch has four meanings at once:

1. software patch: versioning, diffs, commits
2. modular synth patch: signal routing and cables
3. organism/land patch: cultivated growth surface
4. timeline patch: visible history of changes

## 4. Core dramaturgy

The recommended structure is a 3-minute loop using a 5-act form.

This does not mean "Act 1 = dark matter, Act 2 = atoms" in a literal linear way.

Instead:

- the 5 acts are the energetic process
- the natural layers are the material skins
- the side walls preserve previous/future phases
- the floor carries the causal signal
- the rear wall reveals the meta-process

The 5 acts:

```text
I.   Boot / Idle
II.  Feed / Collapse
III. Grow / Morphogenesis
IV.  Corrupt / Optimization
V.   Glitch / Dimension++
```

The classic 5-act structure is adapted for immersive space:

| Dramaturgy | Immersive translation |
|---|---|
| Exposition | spatial orientation, boot, title |
| Rising action | signal intensifies, matter forms |
| Climax | growth over-optimizes and corrupts |
| Falling action | glitch destabilizes the whole room |
| Resolution | recognition of the loop, return to seed |

The ending does not solve the story.

It reveals the rule.

## 7. Motion and pacing principles

The instructor feedback is important: immersive work needs slower movement than a normal 2D screen.

Rules:

1. **Major spatial shifts should take 10–20 seconds.**  
   If focus moves from floor to front wall to side wall, make that wave slow and obvious.

2. **The front wall should carry only one dominant idea at a time.**  
   It can be dense internally, but compositionally it should be readable.

3. **The side walls can carry complexity because the audience chooses to inspect them.**  
   They are good for portals, fragments, histories, and alternate dimensions.

4. **The floor should guide gaze, not demand close reading.**  
   Use large directional motion, not tiny details.

5. **Text must be huge and sparse.**  
   Small code/logs can be texture, but important text must be readable in 2–4 seconds.

6. **Glitch should be structural, not just noisy.**  
   The glitch should reveal the next layer through tearing, compression, overgrowth, or collapse.

7. **No fast strobe unless intentionally limited.**  
   In a large projection room, fast glitch can become visual noise or discomfort.

8. **Make the loop legible through repetition.**  
   Repeat the black seed, graph skeleton, and grow/corrupt/glitch rhythm.

## 10. Sound direction

The soundtrack should be the main temporal driver.

Reference feeling: **Rrose — Purge**.

The important qualities to borrow are not literal imitation, but temporal behavior:

- pulse and drone are inseparable
- rhythm feels almost stationary
- internal texture constantly mutates
- low-end pressure is physical
- repetition is hypnotic but unstable
- the piece escalates by density and pressure, not melody
- there is no cinematic resolution
- there is no clean EDM drop
- the sound feels like a machine-organism breathing

The soundtrack should feel atonal, ritualistic, recursive, industrial, and intelligent.

### Sound structure

| Time | Sound behavior |
|---:|---|
| 0:00–0:20 | sub-bass field, granular noise, sparse pulse |
| 0:20–0:55 | particle clicks, spectral shimmer, metallic microtones |
| 0:55–1:20 | wet organic textures, breathing modulation, insect-like details |
| 1:20–2:15 | factory resonance, pipe pressure, mechanical pulse, low throb |
| 2:15–2:40 | data chatter, broken clocks, digital tearing, compression artifacts |
| 2:40–3:00 | collapse to pressure, return to opening bed |

### AI music prompt

```text
3-minute seamless loop for an immersive projection installation.

Dark atonal experimental techno / electroacoustic soundscape. Inspired by hypnotic minimal techno, psychoacoustic drones, spectral music, machine pulse, and low-frequency physical pressure, but not dancefloor EDM.

The rhythm should feel like pulse and drone are inseparable. The beat should feel almost stationary while internal textures mutate constantly.

Structure:
0:00–0:20 latent cosmic field: sub-bass pressure, granular dark noise, sparse unstable pulses, no melody.
0:20–0:55 matter collapse: tiny particle clicks, metallic chirps, microtonal tones, spectral shimmer, pulse becomes visible.
0:55–1:20 organic morphogenesis: wet granular textures, cellular clicks, breathing modulation, insect-like details, slow pressure waves.
1:20–2:15 infrastructure corruption: pipes, factory resonance, machine rhythm, low-end throb, irregular techno pulse, electrical buzzing, circuit chatter.
2:15–2:40 glitch singularity: dense data pulses, broken clock rhythms, digital tearing, error tones, compression artifacts.
2:40–3:00 collapse back into the opening sub-bass field.

No vocals. No melody. No heroic cinematic climax. No clean EDM drop. The piece should feel like a recursive machine-organism expanding, corrupting, glitching, and being reborn.

Seamless loop. Slow spatial motion. Physical low frequencies. Atonal, ritualistic, recursive, ominous, intelligent.
```

### Visual/audio coupling

- Floor Data should be generated from the soundtrack spectrum.
- The Loop should pulse from low-frequency energy.
- The Patch should update on larger structural pulses, not every beat.
- Meta glitches can be triggered by high-frequency noise bursts.
- Major transitions should happen on pressure changes, not obvious drops.

## 11. Surface-specific script

### Meta script

Meta should mostly stay quiet until the glitch phase, then become more explicit.

Possible chronological progression:

```text

0:20 loading substrate
0:35 parsing audio spectrum
0:55 generating morphogenesis rules
1:20 infrastructure pass started
1:45 optimization loop unstable
2:15 corruption accepted

```

Do not show all of this as explanatory subtitles. Use it as a visual texture with occasional readable lines.

### Data script

Data is the score-body.

It starts as almost invisible low waves.

It becomes:

```text
low pulse
-> spectrum river
-> particle stream
-> organic root flow
-> pipe/circuit current
-> compressed data swarm
-> singularity feed
-> quiet opening pulse
```

Data must always move rear-to-front, except possibly during the final collapse where it folds inward or briefly reverses.

### The Loop script

The Loop receives each major Data pulse and mutates.

```text
black seed
-> particle field
-> molecular cluster
-> cell nucleus
-> root/organism
-> industrial core
-> circuit engine
-> data black hole
-> code/creator eye
-> black seed
```

The Loop is the most readable part of the piece. Do not make it too fragmented until Act V.

### The Patch script

The Patch is a side-wall history.

Each portal/arch can show a phase:

```text
Portal 1: current mutation
Portal 2: previous layer
Portal 3: older organic state
Portal 4: infrastructure state
Portal 5: corrupted variant
```

Alternative chronological arrangement:

```text
near front: latest/current state
middle: previous active states
near rear: oldest/fossilized states and source logs
```

The Patch should make the audience understand that the same story is happening at different scales and different times.
